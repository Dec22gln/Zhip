The Elliptic Curve Cryptography (ECC) used in this app is non-standard and therefore unreliable.

Zilliqa uses Schnorr signatures, which lacked Swift implementation when Zhip was developed. Since Zhip is written in Swift, it was needed to develop Schnorr signatures and other ECC methods from scratch, the result is EllipticCurveKit. It is possible that EllipticCurveKit contains bugs.

However, Zhip and therefor EllipticCurveKit has been tested for over six months and has also gone through a brief review by Zilliqa team members. But this does not guarantee correctness and safety.

Thus, it's recommended to send a very small amount of ZILs to Zhip when starting using it. Verify that you can send those ZILs back to the wallet you sent them from.

If you have any problems sending from Zhip, remember that you can always export your private key and/or keystore from Zhip and import it into any other Zilliqa wallet and access your funds there.

If you find any faults in the source code of either EllipticCurveKit, Zesame (Zilliqa Swift SDK) or Zhip, please report issues on Github and/or fix the bug and submit a pull request.
